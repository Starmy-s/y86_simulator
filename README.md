# 💻 Y86-64 Behavioral Simulator (MetalArch)

本项目是一个基于 **C++17** 实现的 **Y86-64 行为级微处理器模拟器**。设计规范严格遵循经典体系结构教材《CS:APP》（《深入理解计算机系统》）第四章标准，100% 仿真复现了 **SEQ (单周期微架构)** 的硬件控制逻辑与状态机。

项目在软件高层对 CPU 的存储型硬件、组合逻辑网格、系统总线及外部主存实施了等价建模，是研究微架构（Microarchitecture）指令译码、时序控制和硬件级数据流流转的轻量级教学平台。

为保证软件仿真与真实数字电路的状态绝对等价（Functional & State Equivalence），处理器内部元器件在 C++ 代码中实施了严密的强类型抽象与时序对齐：

| 物理硬件组件 | 硬件物理职责 | 本项目 C++ 映射实现方式 | 时序/功能对齐度 |
| --- | --- | --- | --- |
| **通用寄存器堆 (RF)** | 15 个 64 位时序触发器构成的寄存器阵列 | `uint64_t registers[15]` 静态数组 | **100% 时钟沿同步** |
| **程序计数器 (PC)** | 锁存当前/下一条待执行指令内存物理地址的寄存器 | `uint64_t pc` 变量（禁止中途修改，末尾锁存） | **100% 时序对齐** |
| **条件码寄存器 (CC)** | 锁存算术运算状态标志的 3 位触发器阵列（ZF/SF/OF） | `struct ConditionCodes` 状态结构体 | **100% 控制线并联** |
| **算术逻辑单元 (ALU)** | 负责执行加、减、与、异或的核心组合逻辑硬件 | `run_alu(...)` 纯组合逻辑函数 | **功能级 100% 对齐** |
| **多路选择器 (MUX)** | 根据控制信号实现硬件导线选通的数据选择器 | C++ 三目运算符 `? :` 与 `switch-case` 分支 | **100% 选路对齐** |
| **内部数据总线** | 芯片内部传输高低电平信号的铜导线走线 | 阶段函数内部的**局部变量**（随作用域销毁） | **纯抽象瞬时对齐** |
| **主内存 (RAM)** | 位于 CPU 芯片外部、通过总线互联的大容量物理存储 | `std::vector<uint8_t>` 与小端序重组原语 | **边界与端序 90% 对齐** |
| **控制逻辑块 (Control)** | 译码核心组合逻辑，根据指令实时输出使能控制电平 | 与真值表完全一一对应的布尔判断函数（`ctrl_xxx`） | **100% 逻辑表达式对齐** |

## 🗺️ 指令集架构 (ISA) 与机器码结构汇总

Y86-64 采用**变长指令集编码（Variable-length Encoding）**，长度介于 1 字节至 10 字节之间。机器码的第 0 字节高 4 位为指令代码（`icode`），低 4 位为功能代码（`ifun`）。

### 1. 核心指令物理布局

* **`halt` (`00`)**：【1 字节】强制改写处理器状态为 `S_HLT`，晶振停止驱动，全机断电。
* **`nop` (`10`)**：【1 字节】空操作，内部控制线保持全局静默，PC 顺序递增 `pc += 1`。
* **`irmovq V, rB` (`30 f rB [8-Byte V]`)**：【10 字节】将 8 字节立即数 $V$ 送往寄存器 $rB$。由于不使用源寄存器，`rA` 引脚对应机器码置为 `f`（物理断路状态 `R_NONE`）。
* **`addq rA, rB` (`60 rArB`)**：【2 字节】指令控制线 `6` 激活，ALU 加法器通路 `0` 选通，执行寄存器补码加法并写回 $rB$。

### 2. 寄存器引脚物理选通编码

机器码中用 4 位二进制（半字节）作为通用寄存器阵列的物理地址引脚：
`0`➡️`%rax` ｜ `1`➡️`%rcx` ｜ `2`➡️`%rdx` ｜ `3`➡️`%rbx` ｜ `4`➡️`%rsp` (物理堆栈指针) ｜ `f`➡️`R_NONE` (物理断路)

## ⚡ 单周期（SEQ）微架构六阶段执行时序

每一个时钟周期内，电信号在模拟主板上严格遵循以下六个阶段自发流转。以 `addq %rax, %rcx`（机器码：`60 01`）为例：

1. **取指阶段 (Fetch)**：利用当前 `pc` 寻址内存，抓取 `60` 与 `01`。字节拆分器解构出 `icode=6 (I_OPQ)`，`ifun=0 (A_ADD)`，`srcA=0 (R_RAX)`，`srcB=1 (R_RCX)`。计算顺序推进地址 `valP = pc + 2`。
2. **译码阶段 (Decode)**：寄存器堆根据 `srcA` 和 `srcB` 地址引脚，瞬间导出对应的电平数据 `valA <- R[%rax]`，`valB <- R[%rcx]`。
3. **执行阶段 (Execute)**：控制信号激活 ALU，`ifun=0` 打开**加法器硬通路**，执行 `valE = valB + valA`。由于 `set_cc` 使能线拉高，组合逻辑电路计算出下一周期的条件码预写值 `next_cc`。
4. **访存阶段 (Memory)**：由于算术运算指令不涉及内存读写，内存控制块保持静默，读写使能线拉低。
5. **写回阶段 (Writeback)**：多路选择器将写回目标引脚 `dstE` 指向 `rB`（即 `%rcx`），并将计算成果 `valE` 送往该寄存器的预写锁存端。
6. **PC更新阶段 (PC Update)**：在 `clock_tick()` 的末尾触发**时钟上升沿同步交割**。寄存器堆、条件码触发器、PC 寄存器同时合闸锁存：`R[%rcx] <- valE`，`CC <- next_cc`，`pc <- valP`。

## 🚀 编译与运行 (Quick Start)

项目基于标准 CMake 构建，无任何外部库依赖。

### 1. 编译构建

```bash
mkdir build && cd build
cmake ..
cmake --build .

```

### 2. 通电运行指南

```bash
cd build
./y86_sim ./machine_code/asum.yo

```
