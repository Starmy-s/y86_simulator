在体系结构中，按照 CSAPP 的 SEQ（单周期）和 PIPE（流水线）设计规范，要完整跑通 Y86-64 指令集，主板和芯片内部一共包含 **8 大类硬件实体**。

在行为级模拟器（Behavioral Simulator）中，我们追求的是**状态和功能的等价（Functional/State Equivalence）**，而不是晶体管层面的物理等价。

以下是所有硬件类型的完整清单，以及我们在 C++ 代码中**对它们的模拟深度与实现对齐方式**：

---

## 💾 一、 存储型硬件（Storage Elements）

存储硬件在单周期内表现为“期初读、期末写”。我们在 C++ 中通过“类私有成员变量 + 状态缓冲”来达到 **100% 的状态等价模拟**。

### 1. 通用寄存器堆 (Register File)

* **真实硬件**：由 15 个 64 位 D 触发器组（Flip-Flops）构成的硬件阵列，拥有两个独立的读端口（Read Ports）和一个写端口（Write Port），支持在一个周期内同时读取 `rA`、`rB` 并写入 `valE`。
* **模拟深度 (100%)**：在 C++ 中使用一个固定大小的数组模拟：
```cpp
uint64_t registers[15] = {0}; // 映射 %rax 到 %r14

```


* **读模拟**：直接通过数组下标任意读取（等价于多路复用引脚引出电平）。
* **写模拟**：引入 `next_registers` 缓冲，在时钟上升沿一瞬间用 `std::memcpy` 统一刷新，完美模拟**单时钟沿同步写入**。



### 2. 程序计数器 (Program Counter - PC)

* **真实硬件**：一个独立的 64 位时序寄存器，位于取指阶段最前端，其输出端直接连接到指令内存的物理地址译码器上。
* **模拟深度 (100%)**：在 CPU 类中定义为：
```cpp
uint64_t pc = 0;

```


严格禁止在取指阶段直接修改 `pc`。必须通过 `next_pc` 暂存指针，在时钟周期（`clock_tick`）的最后一行代码执行 `pc = next_pc`，精确对齐硬件的锁存时序。

### 3. 条件码寄存器 (Condition Codes - CC)

* **真实硬件**：由 3 个独立的 1 位触发器（ZF-零标志, SF-符号标志, OF-溢出标志）组成的寄存器，只在执行算术逻辑运算（`opq`）时由控制信号决定是否更新。
* **模拟深度 (100%)**：用一个布尔结构体模拟：
```cpp
struct ConditionCodes { bool ZF = 1; bool SF = 0; bool OF = 0; };

```


逻辑完全对齐：只有当控制块发出 `set_cc` 信号时，才会改写下一周期的 CC。

## 🎛️ 二、 计算与组合逻辑硬件（Combinational Hardware）

这些硬件在芯片中由纯粹的门电路（门阵列）构成，没有时钟线，**电信号喂进去立刻吐出结果**。在 C++ 中，我们将其**完全剥离成“纯函数（Pure Functions）”**，**不维护任何内部状态**。

### 4. 算术逻辑单元 (ALU)

* **真实硬件**：由加法器、位移器、与或非门构成的巨型组合逻辑块。输入端接收数据 `A` 和 `B` 以及操作码 `alu_fun`，输出端输出 `valE`，并实时拉出 3根 状态导线送往条件码判定电路。
* **模拟深度 (功能级 100%)**：我们不需要模拟半加器、全加器电路，而是直接利用 CPU 本身的算术运算符进行**数学等价模拟**：
```cpp
uint64_t run_alu(uint8_t alu_fun, uint64_t valA, uint64_t valB) {
    if (alu_fun == F_ADD) return valB + valA; // 利用x86硬件的加法模拟Y86加法
    // ... 
}

```



### 5. 字节拆分器 (Splitter) / 符号扩展器 (Sign Extender)

* **真实硬件**：
* **Splitter**：不占用任何物理门，就是把 8 根总线导线一分为二，4根拉走当控制线，4根拉走当功能线。
* **Sign Extender**：把机器码里抽出来的 4 字节地址或 8 字节立即数，通过硬连线复制最高位（符号位），扩展成标准的 64 位补码。


* **模拟深度 (硬连线等价 100%)**：
* 在 C++ 中通过**位移和掩码**模拟 Splitter：`(code_byte >> 4) & 0x0F`。
* 通过**强类型转换**模拟符号扩展器：直接利用 C++ 的 `int32_t` 自动转 `int64_t` 来利用底层编译器的符号位填充技术。

## 🔌 三、 传输与互联硬件（Interconnect & Bus）

### 6. 数据总线与地址总线 (Buses) / 多路选择器 (MUX)

* **真实硬件**：
* **Buses**：密密麻麻的铜导线，负责在各个阶段之间投递数据电平（如 `valA`, `valB`, `valC`, `valM`, `valP`）。
* **MUX**：硬件开关，根据控制块提供的电平信号，决定让哪一根导线的数据流过（例如：`srcA` 控制块决定 `valA` 是去读 `rA` 还是读 `%rsp`）。


* **模拟深度 (抽象等价 80%)**：
* **总线模拟**：我们**不需要**在 C++ 里声明物理导线对象，而是直接利用局部变量（Local Variables）来扮演总线。局部变量在生命周期结束时自动销毁，完美对应了“电平信号随着时钟周期结束而消失”的物理特性。
* **MUX模拟**：直接用 C++ 的 **三目运算符** 或 `switch-case` 模拟多路开关：
```cpp
uint64_t valA = (srcA != REG_NONE) ? registers[srcA] : 0; // MUX 开关切换

```

## 🏪 四、 系统边界硬件（System Boundaries）

### 7. 物理内存条 (Main Memory / RAM)

* **真实硬件**：坐落在 CPU 外部的主板芯片，通过物理引脚与 CPU 总线连接。它的读写速度比 CPU 慢得多。
* **模拟深度 (黑盒解耦模拟 90%)**：
```cpp
std::vector<uint8_t> storage(4096);

```


* **对齐部分**：我们严格模拟了内存的**边界检查**（越界抛出 `STAT_ADR`）以及小端序（Little-Endian）的字节流拼接拆解（8 字节的长整型如何分布在 8 个连续内存槽位里）。
* **未对齐/忽略部分**：作为单周期行为级模拟，我们**忽略了真实的物理延迟和电荷刷新时间（Latency）**。在代码里，读写内存被视作瞬间完成，这与真实硬件中内存响应慢、需要等待多周期（Wait States）的情况不符（如果要对齐这一层，需要编写复杂的缓存 Cache 模拟器）。

## 🧠 五、 核心指挥官（The Commander）

### 8. 控制逻辑块 (Control Logic Blocks)

* **真实硬件**：这是 CPU 内部最烧脑的纯组合逻辑硬核电门。它们只读取 `icode`, `ifun` 和条件码 `CC`，通过布尔代数化简出来的门电路，实时吐出各种 0 或 1 的使能开关信号（如 `need_valC`, `srcA`, `set_cc`, `instr_valid`）。
* **模拟深度 (100% 逻辑对齐)**：在 C++ 中，我们拒绝编写面条式的过程代码，而是将其写成**与硬件真值表完全一一对应的布尔逻辑函数**：
```cpp
bool ctrl_set_cc(uint8_t icode) {
    return (icode == I_OPQ); // 只有 OPQ 指令才会拉高 set_cc 导线的电平
}

```


这部分在逻辑上与芯片设计手册上的逻辑表达式做到了完全等价。

## 📊 终极盘点：我们在多大程度上进行了模拟？

| 硬件类型 | 真实硬件本质 | C++ 模拟实现手段 | 模拟对齐度 | 为什么留有遗憾？（未模拟的部分） |
| --- | --- | --- | --- | --- |
| **寄存器堆/PC** | D触发器物理阵列 | 类私有数组 + 双缓冲状态覆盖 | **100%** | 无，状态和时序完全等价。 |
| **ALU** | 加法器与门阵列 | C++ 算术运算符（`+`, `-`, `&`, `^`） | **功能级 100%** | 忽略了电信号通过门电路时的**物理延迟（纳秒级毛刺）**。 |
| **多路选择器 (MUX)** | 物理选通门开关 | 三目运算符 (`? :`) 与 `switch` | **100%** | 无，选路逻辑完全对齐。 |
| **主板总线** | 铜导线、PCB走线 | 阶段函数内部的**局部变量** | **纯抽象模拟** | 忽略了导线电容导致的信号衰减。 |
| **物理内存 (RAM)** | 电容阵列/地址译码器 | `std::vector<uint8_t>` + 位移函数 | **90%** | 模拟了大小端和边界，但**把访存当成了瞬间完成**，忽略了内存等待周期。 |
| **控制逻辑块** | 组合逻辑布尔电门 | 纯布尔判断函数 (`ctrl_xxx`) | **100%** | 逻辑表达式与硬件真值表完全一致。 |
